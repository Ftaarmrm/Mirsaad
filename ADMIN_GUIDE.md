# 🛡️ دليل لوحة تحكم الأدمن

## 🔐 الوصول

**الرابط:** `https://yourdomain.com/admin/ads`

أو من الموقع: اذهب لأسفل الصفحة → **"إدارة"** (في الـ footer)

---

## ⚙️ الإعداد الأولي

### 1. إعداد كلمة المرور

في ملف `.env`:

```env
# كلمة مرور قوية للأدمن (غيّرها فوراً!)
ADMIN_PASSWORD="ضع_كلمة_مرور_قوية_هنا"

# مفتاح سري (32 حرف عشوائي)
ADMIN_SECRET="$(openssl rand -hex 32)"
```

### 2. إعداد AdSense Client ID

```env
# من حسابك في AdSense
NEXT_PUBLIC_ADSENSE_CLIENT_ID="ca-pub-1234567890123456"
```

### 3. تحديث قاعدة البيانات

```bash
bun run db:push   # ينشئ جدول AdSlot الجديد
```

### 4. شغّل المشروع

```bash
bun run dev
```

## 🎯 إضافة إعلان

1. اذهب إلى `/admin/ads`
2. أدخل كلمة المرور
3. اضغط **"إضافة إعلان"**
4. املأ النموذج:

| الحقل | الوصف | مثال |
|------|-------|------|
| **الاسم** | اسم وصفي | "بانر الصفحة الرئيسية" |
| **Slot ID** | من AdSense | `1234567890` |
| **الموضع** | أين سيظهر | الصفحة الرئيسية - أعلى |
| **التنسيق** | شكل الإعلان | أفقي (Banner) |
| **العرض** | بالـ px (اختياري) | 728 |
| **الارتفاع** | بالـ px (اختياري) | 90 |
| **متجاوب** | يتكيف مع الشاشة | ✓ |
| **مفعّل** | يظهر على الموقع | ✓ |

5. اضغط **"إنشاء"** ✅

## 📍 المواضع المتاحة

### الصفحة الرئيسية:
- `HOME_TOP` - أعلى الصفحة
- `HOME_MIDDLE` - وسط الصفحة
- `HOME_SIDEBAR` - الشريط الجانبي
- `HOME_BOTTOM` - أسفل الصفحة

### صفحة الأخبار:
- `NEWS_TOP` - أعلى القائمة
- `NEWS_INFEED` - بين المقالات (كل 6 مقالات)
- `NEWS_SIDEBAR` - جانبي
- `NEWS_BOTTOM` - أسفل

### صفحة المقال:
- `ARTICLE_TOP` - أعلى المقال
- `ARTICLE_INLINE` - داخل المحتوى
- `ARTICLE_BOTTOM` - أسفل المقال

### صفحات أخرى:
- `COUNTRY_TOP`, `COUNTRY_BOTTOM`
- `ANALYTICS_TOP`, `ANALYTICS_BOTTOM`
- `FOOTER`, `CUSTOM`

## 🔄 الأولوية (Priority)

إذا كان لديك أكثر من إعلان لنفس الموضع:
- **رقم أعلى = أولوية أعلى**
- يظهر الإعلان ذو الأولوية الأعلى أولاً
- مثال: 10 يظهر قبل 5

## 💡 نصائح للإيرادات

### ✅ المواضع عالية الأداء:
1. **HOME_TOP** - أول ما يراه الزائر
2. **NEWS_INFEED** - بين المقالات (CTR عالي)
3. **ARTICLE_INLINE** - وسط المحتوى
4. **HOME_SIDEBAR** - يبقى مرئياً طويلاً

### ❌ تجنّب:
- إعلانات في الـ admin pages
- إعلانات في الـ login pages
- إعلانات تغطي المحتوى
- أكثر من 3 إعلانات في نفس الصفحة

### 📊 لزيادة CTR:
- استخدم تنسيق **Native (Fluid)** للمحتوى
- ضع إعلانات قرب المحتوى المرتبط
- استخدم **Responsive** للموبايل
- اختبر مواضع مختلفة (A/B Testing)

## 🚨 الأمان

### في الإنتاج:

1. **غيّر كلمة المرور فوراً:**
   ```env
   ADMIN_PASSWORD="كلمة_مرور_معقدة_جداً_و_طويلة"
   ```

2. **مفتاح سري قوي:**
   ```bash
   openssl rand -hex 32
   ```

3. **HTTPS فقط** - تأكد من تشغيل HTTPS

4. **لا تشارك** الـ URL مع أحد

5. **راجع الـ logs** بانتظام للمحاولات المشبوهة

## 🔧 استكشاف الأخطاء

### الإعلان لا يظهر؟

✅ **في التطوير (Development):**
- ستظهر مربعات placeholder ملونة بدلاً من الإعلانات
- **هذا طبيعي!** AdSense لا تعمل على localhost

✅ **في الإنتاج:**
1. تحقق من `NEXT_PUBLIC_ADSENSE_CLIENT_ID` صحيح
2. تحقق من Slot ID صحيح في لوحة الأدمن
3. تحقق من حالة الإعلان (مفعّل ✓)
4. تحقق من حالة AdSense (موافق على الموقع؟)
5. انتظر بعض الوقت (قد يستغرق دقائق للظهور)

### لا أستطيع الدخول للأدمن؟

```bash
# تأكد من ADMIN_PASSWORD مكوّن
grep ADMIN_PASSWORD .env

# إذا لم يكن مكوّناً، أضفه:
echo 'ADMIN_PASSWORD="my-password"' >> .env
echo 'ADMIN_SECRET="'$(openssl rand -hex 32)'"' >> .env

# أعد تشغيل المشروع
```

### نسيت كلمة المرور؟

غيّرها مباشرة في `.env` وأعد تشغيل المشروع.

## 📝 ملاحظات مهمة

### للحصول على Slot ID من AdSense:

1. ادخل [AdSense Dashboard](https://www.google.com/adsense)
2. **Ads** → **By ad unit**
3. اضغط **Create new ad unit**
4. اختر نوع الإعلان (Display, In-feed, In-article, etc.)
5. خصص الإعدادات (المقاس، الاسم، إلخ)
6. انسخ **Ad Slot ID** (الرقم بعد `data-ad-slot=`)
7. الصقه في لوحة الأدمن

### ads.txt:

تأكد من تحديث `public/ads.txt` بـ Publisher ID الصحيح:

```
google.com, pub-1234567890123456, DIRECT, f08c47fec0942fa0
```
